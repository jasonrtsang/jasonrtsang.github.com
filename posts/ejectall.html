Eject All: A physical button to quickly eject all external disks on your Mac
January 1, 2016
This convenient tool in action! Green LED lights on success, red when when not.

Typically, my MacBook lives its life on my desk with a Time Machine (external hard drive) plugged in. Though, when the time comes for it to travel, I always find it a nuisance to manually eject the Time Machine. This is made worse when the login screen is prompted after being inactive or asleep. With this simple Arduino contraption, I can quickly eject the Time Machine, and any other disks plugged-in, with the press of a button; even on the lock screen.

Parts/ Materials:

  • Arduino Uno
  • Breadboard with jump leads
  • Push button
  • LED (2 colours)
  • Resistors

Link to source code/ files

Below is the basic diagram of how the source files interact on OS X. Each box represents a loop, with their executables within.

launchd (.plist)

Python script (.py)

AppleScript (.app)

textfile (.txt)

launchd is a management framework that is open-source and built into OS X. It can schedule and manage user-created processes. In this case, the Python script is our process controlled by launchd. The script simply polls at the Arduino serial port to see if a character is written to it. If true, the script will trigger the AppleScript, which in turn ejects all external disks. The AppleScript also displays a notification with sound, and writes its success to a textfile. The textfile acts as a medium to relay the AppleScript’s output to the Python script. Using the textfile, the Python script writes the value within to the Arduino which in turn flashes the corresponding green or red LED.

There are probably more efficient ways to do this, but judging but its resource usage, it seems to cause no impact to my Macbook’s performance by having a Python script continually poll. Though in terms of deployability, it’s a bit lacking. Every source file has hardcoded dependencies (refer to the GitHub page for the variables). For instance, this tool (as is) will only work if the Arduino is plugged into the same port. In my case, I have it connected to an USB hub along with the Time Drive and several other accessories. Thus, this problem does not affect me. Note that as launchd is used, there is no configurations necessary after the initial setup. Hence, you can disconnect and reconnect the Arduino (to the same port) and the tool will work automatically.

On the occasions I don't have the Arduino connected, I can just run the AppleScript to eject all the disks I have connected. I still find this quicker than manually ejecting. If you wish to only run the AppleScript without the Arduino setup, simply remove the textfile writes by editing the application in Automator.

For the Arduino circuit schematic, it is basically a mashup of the Blink and Button tutorials by Arduino.
